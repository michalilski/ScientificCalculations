Zadanie 2.¶

W zadaniu zostanie sprawdzona zgodno wynikw, przedstawionych na wykresach, w p

Wykres funkcji $f(x) = e^{x}\ln(1+e^{-x})$ w jzyku Python 3 (dla matplotlib, numpy, math)

oraz w programie GeoGebra:

Granica funkcji $f(x): \lim_{x->\infty}f(x) = \frac{\ln(1+e^{x})}{\frac{1}{e^{x}}} \stackrel{H}{=} \lim_{x->\infty}\frac{\frac{-1}{e^{x}+1}}{\frac{-1}{e^{x}}}=\lim_{x->\infty}\frac{e^{x}}{e^{x}+1}=1$

Jak wida na wykresach, w okoliach $x=32$ pojawiaj si oscylacje wok wartoci $1.0$. Od okoo $x=37$ warto wynosi ju stale $0$ co jest niezgodne z obliczon granic. Problemem jest wymnaanie przez siebie wartoci $e^x$, ktra przykadowo dla $x=37$ jest wzgldnie dua, z wartoci $\ln(1+e^{x})$, ktra bardzo szybko zbiega do $0$. Ostatecznie mnoymy jak du warto, przez warto $\approx 0$, co w rezultacie daje $0$, przez wykonane zaokrglenia liczb. Jak wida dla $x=30$ warto funkcji jest w przyblieniu zgodna z oczekiwaniami, a ju dla $x=35$ pojawiaj si znaczce dla bdy. Dla $x>37$ wynik jest cakowicie bdny, co pokazuje e zadanie jest le uwarunkowane.

function matcond(n::Int, c::Float64)
# Function generates a random square matrix A of size n with
# a given condition number c.
# Inputs:
#	n: size of matrix A, n>1

#
# Usage: matcond(10, 100.0)
#
# Pawel Zielinski
        if n < 2
         error("size n should be > 1")
        end
        if c< 1.0
         error("condition number  c of a matrix  should be >= 1.0")
        end
        (U,S,V)=svd(rand(n,n))
        return U*diagm(0 =>[LinRange(1.0,c,n);])*V'

W przypadku macierzy Hilberta $H_n$ oraz losowej macierzy $R_n$ wraz ze wzrostem rozmiaru, wzrasta bd podczas obliczania wartoci $x$. W przypadku macierzy $H_n$ bd ten ronie jednak o wiele szybciej ni w przypadku $R_n$, co jest zgodne z obserwacj wzrostu wskanika uwarunkowania. Wartoci tego wskanika dla macierzy $H_n$ rosn wzgldnie szybko, ponadto wida korelacj midzy wartoci bdu a wartoci wskanika. Porwnujc wyniki dla $H_n$ i $R_n$ mona zauway, e zadanie dla $H_n$ jest le uwarunkowane.
